# MHCAssist - Mental Health Care Act Compliance Registry

MHCAssist is a specialized digital registry designed to streamline compliance with the **South African Mental Health Care Act (MHCA) No. 17 of 2002**. It provides healthcare professionals with tools to manage patient intakes, track mandatory 72-hour observation periods, and generate official regulatory documentation.

## 🚀 Key Features

- **Digital Patient Intake**: Streamlined registration of patients admitted under the MHCA.
- **72-Hour Observation Tracker**: Real-time visual timers for mandatory observation periods with status indicators (Admitted, Observing, Awaiting Review).
- **Automated MHCA-04 Generation**: Instant generation of official PDF forms with integrated:
  - **Dynamic QR Codes**: Scannable links that open a WhatsApp chat with the clinic.
  - **Standard Barcodes**: CODE128 barcodes for facility indexing.
  - **Digital Audit Trail**: Records of SMS and WhatsApp notifications sent to family members.
- **Role-Based Access Control**: Specialized views and permissions for Nurses, Clinicians, Review Boards, and Administrators.
- **Offline-First Capabilities**: Local data persistence for demo modes and unreliable network environments.
- **Real-Time Collaboration**: Powered by Firebase for instant updates across multiple devices.

## 🛠️ Tech Stack

- **Frontend**: React 19, TypeScript, Tailwind CSS
- **Backend/Database**: Firebase (Firestore, Authentication)
- **PDF Generation**: `@react-pdf/renderer`
- **Icons**: Lucide React
- **Animations**: Motion (formerly Framer Motion)

## 🏃 Getting Started

### Prerequisites

- Node.js (v18 or higher)
- npm or yarn

### Installation

1. Clone the repository or download the source code.
2. Install dependencies:
   ```bash
   npm install
   ```

### Running the Application

To start the development server:
```bash
npm run dev
```
The application will be available at `http://localhost:3000`.

### Demo Mode

For quick exploration without setting up a full Firebase backend:
1. Navigate to the Login screen.
2. Click on the **"Nurse"** or **"Clinician"** buttons under the **"Demo Mode"** section.
3. This will use local storage to simulate the application's functionality.

## 📄 Configuration

To use your own Firebase project, update the `firebase-applet-config.json` file in the root directory with your credentials:

```json
{
  "apiKey": "YOUR_API_KEY",
  "authDomain": "YOUR_AUTH_DOMAIN",
  "projectId": "YOUR_PROJECT_ID",
  "appId": "YOUR_APP_ID",
  "firestoreDatabaseId": "(default)"
}
```
